# macOS 和 Linux 系统运行须知

如果您是在 macOS 或者是 Linux 系统上全新下载并准备运行本程序，可能会遇到**没有执行权限**或**系统安全拦截**的问题。这属于操作系统的常规保护机制。
请参考以下步骤进行简单的配置，即可正常使用：

---

## 一、赋予脚本文本执行权限（macOS & Linux 通用）

新下载的脚本文件可能会因为缺少“可执行”权限而无法直接通过双击或命令行启动。解决方法如下：

1. 打开您系统自带的 **终端 (Terminal)**。
2. 在终端内输入 `chmod +x ` **（注意 `+x` 后面必须保留一个空格！）**。
3. 将您要运行的脚本文件（例如 `一键开启_刷步网页版_macOS.command` 或是 Linux 的 `.sh` 脚本）**拖拽**到终端窗口中，文件路径会自动补全。
4. 按下回车键（Enter）。

此时，脚本已经获得了执行权限。

---

## 二、解除 macOS 特有的安全拦截

macOS 对于从浏览器下载的内容会自动附加上隔离属性（`com.apple.quarantine`），并且默认不信任未签名的开发者。您可能会遇到“已损坏，无法打开”或“无法验证开发者”的弹窗。

### 方案 A：开启“任何来源”（一键解决，推荐）
1. 打开 **终端 (Terminal)**。
2. 粘贴以下代码并回车（执行过程中可能需要您输入电脑的**开机密码**，输入时密码不会显示出来，盲输完毕后按回车即可）：
   ```bash
   sudo spctl --master-disable
   ```
3. 打开电脑的 **系统设置 -> 隐私与安全性**，稍微向下滚动，在“安全性”板块下，确保勾选了 **【任何来源】**。
4. 此后即可顺畅双击运行，不再被恶意拦截。

### 方案 B：手动仍然打开（偶尔使用）
1. 尝试双击运行脚本，如果弹窗提示阻止您运行，请点击“取消”或“好”将其关闭。
2. 不要关闭窗口，立刻打开 **系统设置 -> 隐私与安全性**。
3. 稍微往下滚动，您会看到一段提示文本，说明刚才某个程序被阻止运行了。
4. 点击旁边的 **【仍然打开】（或 Open Anyway）**。
5. 再次弹窗确认时，选择 **“打开”** 即可。

按照以上指南操作完成后，您的刷步程序就可以正常启动服务了！
